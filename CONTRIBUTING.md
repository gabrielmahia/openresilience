## How to Contribute

### Reporting Issues
1. Check existing issues first
2. Use descriptive titles
3. Include reproduction steps
4. Specify environment details

### Suggesting Enhancements
1. Open an issue with [ENHANCEMENT] prefix
2. Describe the use case
3. Explain expected behavior
4. Consider data sovereignty implications

### Code Contributions
Due to the NoDerivatives clause, direct code contributions require explicit permission.

For significant changes:
1. Open an issue first to discuss
2. Email contact@aikungfu.dev for collaboration terms
3. Ensure alignment with ethics-first engineering principles

## Development Standards
- Python 3.9+
- Type hints encouraged
- Docstrings for public functions
- Test coverage for logic modules
- No hard-coded credentials
